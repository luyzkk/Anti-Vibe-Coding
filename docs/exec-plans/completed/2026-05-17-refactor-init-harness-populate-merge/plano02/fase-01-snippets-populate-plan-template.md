## Gotchas

- **G2 do plano (filosoficos):** Documentar na lista de Passo 1 que `COMPOUND_ENGINEERING.md` e `PRODUCT_SENSE.md` sao excluidos. A blocklist literal vai para o gerador, nao para o template. Mas o template menciona a exclusao na secao "Notas" para o dev que ler o PLAN.md final entender.
- **G3 do plano (wave-based):** Os comentarios `<!-- wave: 1 -->` e `<!-- wave: 2 -->` sao contrato com o `/execute-plan` (Plano 01 fase-01 validou suporte). NAO renomear para `## Wave 1` — ficam como HTML comment para nao poluir o sumario do plano.
- **G4 do plano (`harness:validate` literal):** Comando exato com `&&` shell, sem array. Caminhos relativos (`scripts/harness-validate.ts`), nunca absolutos. Test #4 da fase-02 verifica string literal.
- **G5 do plano (Windows path-safety):** Placeholder `{{DATE}}` e substituido pelo gerador (fase-02) em formato `YYYY-MM-DDTHH-MM-SSZ` (sem `:`). O caminho final do PLAN.md emitido vira `docs/exec-plans/active/{date}-populate-harness/PLAN.md` — Windows-friendly.
- **G6 do plano (`{{SHARED_GLOSSARY_BLOCK}}` opcional):** Quando undefined, virar string vazia EXATAMENTE — sem deixar linha em branco extra que destruiria o ritmo visual do PLAN.md.
